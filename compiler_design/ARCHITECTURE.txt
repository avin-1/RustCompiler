================================================================================
                    KEY DESIGN DECISIONS
================================================================================

1. SINGLE-PASS COMPILATION
   • All phases happen during parsing
   • Syntax-Directed Translation (SDT)
   • Efficient: O(n) time complexity

2. EMBEDDED HELPERS
   • All code in parser_standalone.y
   • No external .c/.h files
   • Meets "lex and bison only" requirement

3. SIMPLE DATA STRUCTURES
   • Fixed-size arrays (not dynamic)
   • Linear search (adequate for small programs)
   • Easy to understand and debug

4. CLEAR SEPARATION
   • Lexer: tokenization only
   • Parser: everything else
   • Main: orchestration only

5. COMPREHENSIVE OUTPUT
   • Shows all compilation phases
   • Educational value
   • Easy to verify correctness
