================================================================================
🎯 WHAT'S IMPLEMENTED
================================================================================

PHASE 1: LEXICAL ANALYSIS (in lexer.l)
  - Keywords: let, mut, fn, if, else, while, return, i32, f64, bool
  - Operators: + - * / = == != < > <= >=
  - Literals: integers, floats, strings
  - Comments: // and /* */
  - Token output with line numbers

PHASE 2: SYNTAX ANALYSIS (in parser_standalone.y)
  - Context-Free Grammar for Rust subset
  - LALR(1) parsing algorithm
  - Error recovery
  - Syntax validation

PHASE 3: PARSE TREE (in parser_standalone.y)
  - Tree construction during parsing
  - Hierarchical representation
  - File output (parsetree.txt)
  - Memory management

PHASE 4: SEMANTIC ANALYSIS (in parser_standalone.y)
  - Symbol table with 256 entries
  - Variable declaration checking
  - Type checking (i32, f64)
  - Scope management
  - Duplicate declaration detection
  - Undefined variable detection

PHASE 5: INTERMEDIATE CODE (in parser_standalone.y)
  - Three-Address Code (TAC) generation
  - Temporary variable generation (t0, t1, ...)
  - Expression evaluation
  - Control flow handling

PHASE 6: OPTIMIZATION (in parser_standalone.y)
  - Constant folding
  - Compile-time evaluation
  - Example: 3 + 4 * 2 → 11 (computed at compile time)

================================================================================
🏗️ BUILD PROCESS
================================================================================

Step 1: win_flex lexer.l
  → Generates: lex.yy.c (lexer implementation)

Step 2: win_bison -d parser_standalone.y
  → Generates: parser_standalone.tab.c (parser implementation)
  → Generates: parser_standalone.tab.h (token definitions)

Step 3: gcc -o compiler_standalone.exe lex.yy.c parser_standalone.tab.c main_standalone.c
  → Links everything into executable

Result: compiler_standalone.exe

================================================================================
📚 COMPILER THEORY DEMONSTRATED
================================================================================

1. REGULAR EXPRESSIONS → DFA
   - Lexer uses Flex-generated DFA for tokenization
   - Efficient O(n) scanning

2. CONTEXT-FREE GRAMMAR → LALR(1) PARSER
   - Parser uses Bison-generated LALR(1) parser
   - Handles operator precedence and associativity

3. SYNTAX-DIRECTED TRANSLATION
   - Semantic actions execute during parsing
   - Single-pass compilation

4. SYMBOL TABLE
   - Hash table / linear search
   - Scope management
   - Type information storage

5. INTERMEDIATE REPRESENTATION
   - Three-address code
   - Temporary variables
   - Machine-independent

6. CODE OPTIMIZATION
   - Constant folding
   - Compile-time evaluation
   - Dead code elimination (basic)
